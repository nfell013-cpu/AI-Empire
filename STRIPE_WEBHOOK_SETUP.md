# Stripe Webhook Configuration Guide

## Overview

AI Empire uses Stripe webhooks to automatically process payments, activate subscriptions, credit tokens, and handle cancellations for all 45 AI tools.

---

## Webhook Endpoint URL

```
https://YOUR_DOMAIN/api/stripe/webhook
```

**Examples:**
- Production: `https://aiempire.com/api/stripe/webhook`
- Staging: `https://staging.aiempire.com/api/stripe/webhook`
- Local (via Stripe CLI): `http://localhost:3000/api/stripe/webhook`

---

## Required Webhook Events

Configure your Stripe webhook to listen for these events:

| Event | Purpose |
|-------|---------|
| `checkout.session.completed` | Handles: token pack purchases, app subscriptions, tiered plan activations, ad purchases, legacy one-time payments |
| `invoice.payment_succeeded` | Handles: monthly subscription renewals for all 45 apps and tiered plans |
| `customer.subscription.deleted` | Handles: subscription cancellations — deactivates access |

### Optional (recommended for monitoring):

| Event | Purpose |
|-------|---------|
| `invoice.payment_failed` | Alert on failed renewal payments |
| `customer.subscription.updated` | Track plan changes / upgrades |
| `charge.refunded` | Handle refund processing |

## How to Add Webhook in Stripe Dashboard

### Step 1: Open Stripe Dashboard
Navigate to **[https://dashboard.stripe.com/webhooks](https://dashboard.stripe.com/webhooks)**

### Step 2: Add Endpoint
1. Click **"Add endpoint"**
2. Enter your webhook URL: `https://YOUR_DOMAIN/api/stripe/webhook`
3. Under **"Select events to listen to"**, click **"Select events"**
4. Search and add:
   - `checkout.session.completed`
   - `invoice.payment_succeeded`
   - `customer.subscription.deleted`
5. Click **"Add endpoint"**

### Step 3: Get Webhook Signing Secret
1. After creating the endpoint, click on it
2. Click **"Reveal"** next to "Signing secret"
3. Copy the secret (starts with `whsec_`)
4. Add it to your `.env` file:

```env
STRIPE_WEBHOOK_SECRET=whsec_YOUR_SIGNING_SECRET_HERE
```

## Local Development with Stripe CLI

### Install Stripe CLI
```bash
# macOS
brew install stripe/stripe-cli/stripe

# Linux
curl -s https://packages.stripe.dev/api/security/keypair/stripe-cli-gpg/public | gpg --dearmor | sudo tee /usr/share/keyrings/stripe.gpg
echo "deb [signed-by=/usr/share/keyrings/stripe.gpg] https://packages.stripe.dev/stripe-cli-debian-local stable main" | sudo tee -a /etc/apt/sources.list.d/stripe.list
sudo apt update && sudo apt install stripe
```

### Forward Events to Local Server
```bash
stripe login
stripe listen --forward-to localhost:3000/api/stripe/webhook
```

The CLI will output a webhook signing secret — use it in your local `.env`:
```
STRIPE_WEBHOOK_SECRET=whsec_xxxxx
```

### Trigger Test Events
```bash
# Test a successful checkout
stripe trigger checkout.session.completed

# Test a subscription renewal
stripe trigger invoice.payment_succeeded

# Test a cancellation
stripe trigger customer.subscription.deleted
```

## What the Webhook Handles

### Token Purchases (`type: "token_purchase"`)
- Credits tokens to user's balance via `addTokens()`
- Updates `TokenPurchase` record status to "completed"
- Logged in `TokenTransaction` table

### App Subscriptions (`type: "app_subscription"`)
- Stores in `User.subscriptions` JSON: `{ "toolSlug": { active: true, subId, expiresAt } }`
- Also updates legacy per-field columns for backward compatibility
- All 45 apps use the same generic handler

### Tiered Plans (`type: "tiered_plan"`)
- Stores in `User.subscriptions` JSON under `_plan` key
- Tracks `planId`, `appsIncluded`, `subId`, `expiresAt`

### Ad Purchases (`productType: "ad_purchase"`)
- Updates `AdPayment` status
- Sends admin notification via SMS/email

### Subscription Renewals (`invoice.payment_succeeded`)
- Extends `expiresAt` by 1 month for matched subscription

### Cancellations (`customer.subscription.deleted`)
- Sets subscription `active: false` in the JSON field
- Clears legacy per-field subscription data

## Troubleshooting

### Webhook returns 400 (Invalid signature)
- Verify `STRIPE_WEBHOOK_SECRET` matches the signing secret in Stripe Dashboard
- Ensure the raw request body is being passed (not parsed JSON)

### Tokens not credited after purchase
- Check webhook logs in Stripe Dashboard → Webhooks → endpoint → Events
- Verify `type: "token_purchase"` exists in checkout session metadata
- Check `TokenPurchase` and `TokenTransaction` tables in database

### Subscription not activating
- Verify `userId` and `toolSlug` in checkout session metadata
- Check `User.subscriptions` JSON field in database
- Review server logs for webhook processing errors

### Events not arriving
- Verify endpoint URL is accessible from the internet
- Check that correct events are selected in webhook configuration
- For local dev, ensure `stripe listen` is running
